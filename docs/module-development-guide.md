# Module Development Guide

How to add a new Tier 1 (CDK-native) module to the Quick Suite Starter Kit.

For external accelerators (Tier 2), see the [Accelerator Integration Guide](accelerator-integration-guide.md).

## Quick checklist

1. Create the module directory
2. Write `config.yaml`
3. Write the CDK construct
4. Write the Terraform wrapper
5. Register in `app.ts`
6. Register in `synth-cfn-template.ts`
7. Write tests
8. Update the manifest template
9. Update docs

## Step-by-step

### 1. Create the module directory

Pick the right parent based on what the module does:

```
governance/my-module/       # IAM, permissions, subscription management
data-sources/my-source/     # QuickSight data source integrations
core/my-utility/            # Shared infrastructure (custom resources, constructs)
```

### 2. Write config.yaml

```yaml
name: my-module
version: 0.1.0
status: preview              # ready | preview | planned
iac_type: cdk
dependencies:
  - governance/subscription  # modules that must deploy before this one
params:
  - name: my_required_param
    required: true
    type: string
    pattern: "^[a-zA-Z0-9-]+$"
  - name: my_optional_param
    required: false
    default: "some-default"
    type: string
artifacts:
  - cfn-template.yaml
  - terraform/main.tf
```

Supported param types: `string`, `arn`, `email`, `json`, `aws_region`.

### 3. Write the CDK construct

Create `cdk/index.ts` with a Construct class:

```typescript
import { Construct } from 'constructs';

export interface MyModuleProps {
  myRequiredParam: string;
  myOptionalParam?: string;
}

export class MyModule extends Construct {
  constructor(scope: Construct, id: string, props: MyModuleProps) {
    super(scope, id);
    // Your CDK resources here
  }
}
```

Use shared constructs from `core/constructs/` where possible:

```typescript
import { PythonCustomResource } from '../../../core/constructs';
```

### 4. Write the Terraform wrapper

Create `terraform/main.tf` — a thin wrapper around the CFN template:

```hcl
variable "my_required_param" { type = string }
variable "my_optional_param" { type = string; default = "some-default" }

resource "aws_cloudformation_stack" "my_module" {
  name          = "my-module"
  template_body = file("${path.module}/../cfn-template.yaml")
  capabilities  = ["CAPABILITY_IAM", "CAPABILITY_NAMED_IAM"]

  parameters = {
    MyRequiredParam = var.my_required_param
    MyOptionalParam = var.my_optional_param
  }
}
```

The `cfn-template.yaml` is auto-generated by CI — you don't create it manually.

### 5. Register in app.ts

Add your construct to `templates/customer-project/cdk-app/bin/app.ts`:

```typescript
// Add import at the top
import { MyModule } from '../../governance/my-module/cdk';

// Add entry to MODULE_REGISTRY
'governance/my-module': {
  requiredParams: ['my_required_param'],
  create: (stack, params) => {
    new MyModule(stack, 'MyModule', {
      myRequiredParam: params.my_required_param,
      myOptionalParam: params.my_optional_param,
    });
  },
},
```

### 6. Register in synth-cfn-template.ts

Add a case to `core/synth-cfn-template.ts`:

```typescript
// Add import at the top
import { MyModule } from '../governance/my-module/cdk';

// Add case in the switch
case 'governance/my-module':
  new MyModule(stack, 'MyModule', {
    myRequiredParam: cfnParams['my_required_param'].valueAsString,
    myOptionalParam: cfnParams['my_optional_param']?.valueAsString,
  });
  break;
```

### 7. Write tests

Create a CDK assertion test at `governance/my-module/cdk/index.test.ts`:

```typescript
import { App, Stack } from 'aws-cdk-lib';
import { Template } from 'aws-cdk-lib/assertions';
import { MyModule } from './index';

describe('MyModule', () => {
  test('creates expected resources', () => {
    const app = new App();
    const stack = new Stack(app, 'TestStack');
    new MyModule(stack, 'Test', { myRequiredParam: 'test-value' });
    const template = Template.fromStack(stack);
    // Assert your resources exist
  });
});
```

Run with `npm test`.

### 8. Update the manifest template

Add your module (commented out) to both manifest files:

- `manifest.yaml` (root)
- `templates/customer-project/manifest.yaml`

```yaml
modules:
  - governance/subscription                              # ready
  # - governance/my-module                               # preview
```

### 9. Write a README

Create `governance/my-module/README.md`:

```markdown
# Governance: My Module

What this module does and why.

## Prerequisites
- governance/subscription deployed

## Manifest configuration
...

## Architecture
...
```

## Module status lifecycle

- `planned` → you've created the directory and config.yaml but no implementation yet. The orchestrator blocks deployment.
- `preview` → construct exists and deploys, but not production-hardened. The orchestrator warns during validation.
- `ready` → fully implemented, tested, and documented. Safe for customer deployments.

## Testing your module locally

```bash
# Validate the manifest with your module enabled
python3 core/utils/orchestrator.py validate

# Synthesize the CFN template
./synth-cfn.sh governance/my-module

# Run CDK diff to preview changes
npx cdk diff

# Deploy
./deploy.sh
```
